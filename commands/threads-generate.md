스윙크루 원고를 기반으로 다양한 타입의 Threads 게시글을 생성하여 노션 DB에 저장한다.

## 게시글 타입

| 타입 | 설명 | 글자수 | 원고당 개수 |
|------|------|--------|------------|
| 숏폼 | 짧은 팁/훅 | 200~500자 | 5~10개 |
| 롱폼 | 에세이 스토리텔링 | 1,500~3,000자 | 1개 |
| 체인 | 3~5파트 연결 글타래 | 각 200~400자 | 1세트 |

## 실행 순서

### 1. 원고 선택
- 인자가 없으면 노션 "2026년 스윙크루 원고" DB에서 최근 원고 목록을 보여주고 선택하게 한다.
- 인자가 있으면 해당 키워드로 원고를 검색한다.

### 2. 원고 분석
선택된 원고를 fetch하여 전체 내용을 읽고, 게시글로 만들 수 있는 **모든 각도**를 추출한다:

- 핵심 레슨/팁 (각각 별도 게시글)
- 놀라운 사실/통계
- 공감 포인트/흔한 실수
- 프로와 아마추어 차이
- 실전 드릴/연습법
- 비하인드 스토리/에피소드
- Q&A 형식 변환
- 명언/한줄 요약

### 3. 대량 생성 — 3가지 타입 모두 생성

#### 타입 A: 숏폼 (5~10개)
추출한 각도별로 각각 다른 템플릿(꿀팁형, 스토리형, 리스트형, 질문형)을 적용.
- 500자 이내
- 서로 다른 각도/관점, 서로 다른 훅(첫 줄)

#### 타입 B: 롱폼 (1개)
원고 전체 내용을 1,500~3,000자 에세이로 재구성.
- 강력한 훅 → 공감 유도 → 스토리텔링 본론 → 실천 메시지 결론
- 원고의 깊이와 맥락을 살려서 작성
- 약간 정중한 반말 ("~거든요", "~더라고요")

#### 타입 C: 체인 (1세트, 3~5파트)
원고 내용을 연결 글타래로 구성.
- 각 파트 200~400자
- 파트 끝에 "1/n", "2/n" 번호 표기
- 첫 파트: 훅 + 문제 제기 (클릭 유도)
- 중간 파트: 핵심 설명
- 마지막 파트: 결론 + CTA

### 4. 노션 DB 저장 (사용자 확인 없이 자동 진행)
생성된 모든 게시글을 **사용자 확인 없이** 바로 노션 "2026년 Threads 게시글" DB(collection://bb743e71-955f-4909-9f44-2fd97cf58101)에 일괄 저장한다.

#### 숏폼 저장
- 제목: "[원고제목] #번호 - 간략주제"
- 페이지 본문(content): 생성된 Threads 글
- 게시 타입: "숏폼"
- 원본 원고: 원본 페이지 릴레이션
- 게시 상태: "대기"

#### 롱폼 저장
- 제목: "[원고제목] - 롱폼"
- 페이지 본문(content): 에세이 전문
- 게시 타입: "롱폼"
- 원본 원고: 원본 페이지 릴레이션
- 게시 상태: "대기"

#### 체인 저장
- 제목: "[원고제목] - 체인"
- 페이지 본문(content): 모든 파트를 구분선(---)으로 나누어 저장
  ```
  [파트1 내용]  1/3

  ---

  [파트2 내용]  2/3

  ---

  [파트3 내용]  3/3
  ```
- 게시 타입: "체인"
- 원본 원고: 원본 페이지 릴레이션
- 게시 상태: "대기"

### 5. 안내
"노션에서 마음에 드는 글의 '게시 승인' 버튼을 클릭해주세요. 하루 5번 자동으로 Threads에 게시됩니다."
